# Git 版本库与暂存区核心总结

1. **三大核心区域**

   

   - 工作区：本地编辑文件的普通目录；
   - 版本库：目录下的 `.git` 隐藏文件夹，核心包含**暂存区（stage/index）**、Git 自动创建的主分支 `master`、指向当前分支的指针 `HEAD`。

   

2. **文件提交核心流程（两步法）**

   

   - `git add`：把工作区的**文件修改 / 新增文件**添加到**暂存区**；
   - `git commit -m "备注"`：将暂存区的所有修改，一次性提交到当前分支（默认 `master`）。

   

3. **状态查看与结果**

   

   - `git status`：实时查看状态（未暂存的修改、未跟踪的新文件、暂存区待提交内容）；
   - 提交完成后，**暂存区清空**，工作区变为「干净」状态。
