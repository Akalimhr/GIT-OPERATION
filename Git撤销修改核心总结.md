# Git 撤销修改核心总结

针对修改的**三种不同阶段**，Git 提供精准撤销方案，核心命令及用法：

1. 仅修改工作区（未 git add）

   命令：

   ```
   git checkout -- 文件名
   ```

   作用：丢弃工作区修改，恢复到最近一次提交 / 暂存后的状态（-- 必须加，否则是切换分支）。

2. 修改已入暂存区（已 git add，未 git commit）

   两步撤销：

   ① 

   ```
   git reset HEAD 文件名
   ```

   ：把暂存区修改撤回工作区；

   ② 再用 

   ```
   git checkout -- 文件名
   ```

    丢弃工作区修改。

3. 已提交到版本库（已 git commit，未推远程）

   方案：执行版本回退，恢复上一正确版本；

   ⚠️ 注意：若已推送到远程仓库，无法安全撤销。
