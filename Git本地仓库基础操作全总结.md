# Git 本地仓库基础操作全总结

本文汇总了在与豆包千问对话中涉及到的知识点（由AI总结）

**一句话结论：**
你完全可以像平时用电脑一样随意创建和手动删除文件，只要在最后执行 `git add .` （注意点号，代表所有变化，包括删除）的时候，Git 就会自动帮你把这些变动（新增的、删除的、修改的）全部抓取到暂存区。

------

## 一、核心基础概念

1. Git 仓库（版本库 / Repository）

   一个被 Git 全权管理的文件夹，Git 会跟踪文件夹内所有文件的修改、删除、新增，支持版本回溯；仓库本质是文件夹内自动生成的 .git 隐藏文件夹（存储所有版本历史、配置、日志）。

2. 工作区

   你本地编辑文件的文件夹（非.git 部分）。

3. 三层写入机制（Git 提交核心）

   - 本地文件写入：手动编辑并保存文件
   - 暂存区写入：`git add` 命令（临时存储待提交的文件）
   - 版本库写入：`git commit` 命令（永久保存版本）

   

------

## 二、创建 Git 本地仓库（完整步骤）

### 前置条件

已安装 Git，打开 **Git Bash（Windows）/ 终端（Mac）**

### 新建仓库文件夹→进入文件夹（必须执行）→ 验证当前路径（确认位置正确）→初始化 Git 仓库（核心命令）

```
# 新建文件夹（名称自定义，如learngit）
mkdir 文件夹名
cd 文件夹名
pwd
git init
```

✅ 成功提示：`Initialized empty Git repository in xxx/.git/`

⚠️ 注意：`git init` 仅执行**一次**，无需重复初始化。

## 三、终端进入目标目录（核心操作）

###  核心命令

```
cd 文件夹完整路径
```

Windows：直接复制文件夹地址栏路径（如 `cd D:\code\learngit`）。

可以先用 `pwd` 确认当前路径，然后用 `git status` 命令检查当前目录是否是一个 Git 仓库。如果提示 `fatal: not a git repository`，说明你不在一个 Git 仓库内。

## 四、文件创建与本地写入

### 创建文件（以 `readme.md` 为例）

手动在仓库文件夹内新建文件，重命名为 `readme.md`。或代码创建，手动输入内容或代码输入

```
# 命令行创建空文件
touch readme.md
# 命令行直接写入内容
echo "# 我的Git仓库" > readme.md
```

## 五、Git 文件提交核心流程

### 1. 添加文件到暂存区（临时写入）

```
# 添加单个文件
git add readme.md

# 添加当前目录所有文件（最常用）
git add .
```

作用：将文件修改写入 Git 暂存区，标记为待提交状态。

#### 2.提交到版本库（永久写入）

```
git commit -m "提交说明（必填，不能为空）"
```

✅ 示例：`git commit -m "首次提交：新增readme.md文件"`

作用：将暂存区内容永久保存到 Git 版本库，生成不可修改的版本记录。

#### 3.查看历史提交

```
git log
```

## 六、新手必看避坑指南

1. 路径规范（Windows 重点）

   仓库路径禁止包含中文、空格、特殊符号，否则会导致报错、乱码。

2. .git 文件夹禁止操作

   不要手动修改、删除 .git 隐藏文件夹，会导致仓库损坏、历史丢失。

3. 提交备注禁止为空

   git commit -m""引号内必须填写说明文字，不能为空。

4. 执行顺序不可乱

   必须：创建文件 → git add → git commit，无文件无法执行 add。

5. 禁止嵌套初始化

   不要在已存在 Git 仓库的文件夹内重复执行 git init。
